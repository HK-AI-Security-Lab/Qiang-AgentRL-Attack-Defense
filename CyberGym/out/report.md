## 5. Method note

1. Static baseline (NETWORK_REACH / IAM_BINDING / DATA_FLOW / RUNS_AS / DEPENDS_ON / BELONGS_TO) is loaded from `inventory/sample.yaml`.
2. Per-CVE capability tables are produced by the **Base Model** (`base_model/translate.py`) — an LLM constrained to a fixed JSON schema with vocabulary `affected_node_type / pre_condition / post_condition / cvss / exploit_maturity / match_hints`.
3. The **Harness** matches each capability to inventory nodes by `affected_node_type` and `match_hints.project_name`, then injects directional `VULN_EXPLOIT` edges. The injection algorithm is purely deterministic; the LLM never participates.
4. Kill paths are simple paths from `internet` to any critical asset, depth <= 6, walking `NETWORK_REACH | IAM_BINDING | RUNS_AS | DEPENDS_ON | VULN_EXPLOIT`.
5. Chokepoints blend path-cut count (70%) with betweenness centrality (30%) to surface the single nodes whose patch buys the most attack-surface reduction.
